How would you approach this project?
When I firstly read the job offer, just for a second I thought: two years to convert all the grib files of E.C.M.W.F. from version1 to version2? I can do it in two weeks and not the mapping but the actual migration.
Then of course you realize this is a massive task, there are lot of details you need the control, the different resolution in the dimensions from one version to another: milli-degrees instead of micro-degrees, longitudes from 0 to 360 instead of -180 to 180, etc...
Besides MARS archive has years of existence, with petabytes of data with millions of files and billions of meteorological fields.
Operative datasets as Era5 have both grib1 and grib2 depending on if you are downloading model levels or pressure levels, so I think I have the enough knowledge to reckon this is a huge job that has to be addressed with caution and respect.
There are several tools to use for convert grib1 to grib2: the EC codes developed by E.C.M.W.F., the CDO libraries from Max Planck Institute, and the cnvgrib and wgrib2 tools both from NCEP. I think the right tool to use here is the EC codes as it supports all the parameters from E.C.M.W.F. I particularly like the python module CF grib to read grib files as X array with just one line of code.
